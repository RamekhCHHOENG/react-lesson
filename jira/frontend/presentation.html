What is React?

A JavaScript library for building user interfaces

⚛️ Core Idea
React lets you build UIs from small, reusable pieces called components. Each component manages its own state and renders HTML based on that state.
🌟 Key Principles
  • Declarative — describe what UI should look like, not how
  • Component-Based — encapsulated, reusable pieces
  • Virtual DOM — efficient diffing & updates
  • Unidirectional data flow — data flows down
📊 Virtual DOM Flow
State Change
→
New VDOM
→
Diff
→
Real DOM Update

React Hooks

Functions that let you "hook into" React state and lifecycle

useState
Local component state — re-renders on change
useEffect
Side effects — fetch data, subscriptions, timers
useCallback
Memoize functions — avoid unnecessary re-renders
useMemo
Memoize values — expensive computations
useRef
DOM references & mutable values without re-render
useContext
Consume React context — avoid prop drilling
useReducer
Complex state logic with dispatch/action pattern

React Router v7

Client-side routing with nested layouts

🗺️ Route Structure
  • / — Dashboard overview
  • /board — Kanban board view
  • /backlog — Backlog management
  • /projects — Project list
  • /projects/:id — Project detail
  • /issues/:key — Issue detail drawer
🔒 Protected Routes
Routes are wrapped in an AuthLayout component that checks for a valid JWT token. Unauthenticated users are redirected to login.
📝 Route Configuration
const router = createBrowserRouter([ { path: '/', element: <AuthLayout />, children: [ { element: <DashboardLayout />, children: [ { index: true, element: <Dashboard /> }, { path: 'board', element: <Board /> }, { path: 'backlog', element: <Backlog /> }, { path: 'projects', element: <Projects /> }, { path: 'projects/:projectId', element: <ProjectDetail /> }, ] } ] }, { path: '/login', element: <Login /> }, ]);

State Management

Separating server state from client state

🔄 React Query — Server State
  • useQuery — fetch & cache server data
  • useMutation — create, update, delete
  • Optimistic updates — instant UI feedback
  • Auto refetch — keep data fresh
  • Cache invalidation — sync after mutations
🐻 Zustand — Client State
  • UI state — sidebar open/closed, theme
  • Auth state — current user, token
  • Filter state — board/backlog selections
📝 React Query Pattern
// useIssues.ts export function useIssues(projectId?: string) { return useQuery({ queryKey: ['issues', projectId], queryFn: () => fetchIssues(projectId), staleTime: 30_000, }); } // useUpdateIssue.ts export function useUpdateIssue() { const queryClient = useQueryClient(); return useMutation({ mutationFn: updateIssue, onSuccess: () => { queryClient.invalidateQueries({ queryKey: ['issues'] }); }, }); }

Forms & Validation

React Hook Form + Zod schema validation

📝 React Hook Form
  • Uncontrolled inputs — minimal re-renders
  • register() — connects inputs to form
  • handleSubmit() — validates before submit
  • formState.errors — per-field error messages
  • watch() — observe field value changes
��️ Zod Validation
  • Schema-first — define shape & rules
  • Type inference — auto TypeScript type
  • Custom messages — user-friendly errors
📝 Issue Form Example
const issueSchema = z.object({ title: z.string().min(3, 'Title too short'), type: z.enum(['story','task','bug','epic']), priority: z.enum(['highest','high','medium','low']), assignee: z.string().optional(), dueDate: z.string().optional(), }); type IssueFormData = z.infer<typeof issueSchema>; function IssueForm() { const { register, handleSubmit, formState } = useForm<IssueFormData>({ resolver: zodResolver(issueSchema) }); const onSubmit = (data: IssueFormData) => { createIssue.mutate(data); }; return ( <form onSubmit={handleSubmit(onSubmit)}> <input {...register('title')} /> {formState.errors.title && ( <p>{formState.errors.title.message}</p> )} </form> ); }
